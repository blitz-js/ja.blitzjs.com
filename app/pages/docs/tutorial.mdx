このチュートリアルでは、基本的な投票アプリケーションの作成方法を説明します
。

ここでは、すでに Blitz がインストールされている(
[Blitz のインストール方法](./get-started]) を参照ください)ことを前提としま
す。Blitz がインストールされているかどうかは、ターミナルで以下のコマンドを
実行することでわかります。

```sh
blitz -v
```

Blitz がインストールされていれば、そのバージョンが表示されるはずです。イン
ストールされていない場合は、"command not found: blitz "のようなエラーが表
示されます。

## 新しい Blitz アプリの作成 {#creating-a-new-app}

ターミナルで`cd` コマンドを用い、アプリを作成したいフォルダに移動し、次の
コマンドを実行します。

```sh
blitz new my-blitz-app
```

Blitz は現在のフォルダ内に `my-blitz-app` フォルダを作成します。フォームラ
イブラリを選択するように促されます。このチュートリアルでは、推奨されるライ
ブラリ `React Final Form` を選択します。

それでは、`blitz new`が作成したものを見てみましょう。

これらのファイルについて説明します。

- `app/` ディレクトリは、ページや API ルートなど、プロジェクトの多くのもの
  が配置される場所です。

- `app/pages/` ディレクトリは、主要なページフォルダです。Next.js を使った
  ことがある人なら、この違いにすぐに気づくでしょう。Blitz では多く
  の`pages`フォルダを持つことができ、それらはビルド時にマージされます。

- `app/core/` ディレクトリは、アプリ全体で使用されるコンポーネントやフック
  などを置く主な場所です。

- `db/` ディレクトリは、データベースの設定を置く場所です。モデルを書いたり
  、マイグレーションをチェックしたりする場合は、ここに行きます。

- `public/` ディレクトリは、静的アセットを置く場所です。アプリで使用する画
  像・ファイル・動画がある場合は、ここに配置します。

- `.babelrc.js`, `.env`など (つまり、ドットから始まるファイル) は
  、JavaScript ツールの様々な設定ファイルです。

- `blitz.config.js` は、Blitz の高度なカスタム設定を行うためのファイルです
  。フォーマットは `next.config.js` と同じです。

- `tsconfig.json` は、 TypeScript の推奨設定ファイルです。

## ユーザーとして登録する {#sign-up-as-a-user}

Blitz アプリは、ユーザーの Sign Up(会員登録)やログインがすでに設定されてい
る状態で作成されています。早速試してみましょう。**Sign Up** ボタンをクリッ
クします。任意のメールアドレスとパスワードを入力して、**Create Account**を
クリックします。そうすると、あなたのユーザ `id` と `role` が表示されるホー
ムページにリダイレクトされます。

必要であれば、ログアウトしてからログインし直すこともできます。あるいは、ロ
グインページの**Forgot your password?** をクリックして、その流れを試してみ
てください。

## データベースの設定 {#database-setup}

嬉しいことに、SQLite を用いたデータベースの設定がすでに作成されています。
ターミナルで `blitz prisma studio` を実行すると、データベース内のデータを
見ることができる Web インターフェースが開きます。

最初の本格的なプロジェクトを始めるときには、PostgreSQL のようなスケーラブ
ルなデータベースを使用した方が良いでしょう。詳細については
、[データベースの概要](database-overview) を参照してください。今のところ、
デフォルトの SQLite データベースを使用します。

## モデルのためのコード自動生成 {#scaffolding-code-for-our-models}

Blitz には [`generate`](./cli-generate) という便利な CLI コマンドがありま
す。ここでは、`generate`を使って `Question`（質問）と `Choice`（選択肢）
の 2 つのモデルを作成します。 `Question` は質問のテキストと選択肢のリスト
を持ちます。選択肢には、選択肢のテキスト、投票数、関連する質問が含まれます
。Blitz は両方のモデルについて、ID、作成タイムスタンプ、最終更新タイムスタ
ンプを自動的に生成します。

#### Step1: `Question` モデルに関連する生成

```sh
blitz generate all question text:string
```

ターミナルが表示されたら、**Enter** を押して `prisma migrate` （マイグレー
ション） を実行してください。名前を聞かれるので、"add question" と入力して
ください。

```

CREATE    app/pages/questions/[questionId]/edit.tsx
CREATE    app/pages/questions/index.tsx
CREATE    app/pages/questions/new.tsx
CREATE    app/questions/components/QuestionForm.tsx
CREATE    app/questions/queries/getQuestion.ts
CREATE    app/questions/queries/getQuestions.ts
CREATE    app/questions/mutations/createQuestion.ts
CREATE    app/questions/mutations/deleteQuestion.ts
CREATE    app/questions/mutations/updateQuestion.ts

>   id        Int      @default(autoincrement()) @id
>   createdAt DateTime @default(now())
>   updatedAt DateTime @updatedAt
>   text      String
> }

? Run 'prisma migrate dev' to update your database? (Y/n) › true
```


`generate` コマンドに `all` 型を指定すると、モデルとクエリ、Mutation、ペー
ジファイルが生成されます。利用可能な型のオプションの一覧は
[Blitz generate](./cli-generate) のページを参照してください。

#### Step2: 対応するクエリと Mutation を持つ `Choice` モデルの生成

今回は `Choice` モデルのページを生成する必要がないので、`resource` の型を
渡すことにします。

```sh
blitz generate resource choice text votes:int:default=0 belongsTo:question
```

マイグレーションを実行するように促されたら、再度 **Enter** を押し、マイグ
レーションの名前を入力します。

```
CREATE    app/choices/queries/getChoice.ts
CREATE    app/choices/queries/getChoices.ts
CREATE    app/choices/mutations/createChoice.ts
CREATE    app/choices/mutations/deleteChoice.ts
CREATE    app/choices/mutations/updateChoice.ts

✔ Model for 'choice' created in schema.prisma:

> model Choice {
>   id         Int      @default(autoincrement()) @id
>   createdAt  DateTime @default(now())
>   updatedAt  DateTime @updatedAt
>   text       String
>   votes      Int      @default(0)
>   question   Question @relation(fields: [questionId], references: [id])
>   questionId Int
> }

? Run 'prisma migrate dev' to update your database? (Y/n) › true
```

#### Step3: `Question` モデルを更新して `Choice` との関係を持たせる

`db/schema.prisma` を開き、`Question` モデルに `choices Choice[]` を追加し
ます。

```diff
model Question {
  id        Int      @id @default(autoincrement())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  text      String
+ choices   Choice[]
}
```

そして、`blitz prisma generate`を実行して、スキーマの変更に合わせて prisma
クライアントを更新します。これはデータベースの `Question` モデルに実際のフ
ィールドが追加されていないため、データベースのマイグレーションを必要としま
せん。

## モデル属性用に生成されたコードを更新 {#update-generated-code-for-our-model-attributes}

アプリを再度実行する前に、生成されたコードの一部をカスタマイズする必要があります。最終的には、これらの修正は必要ありませんが、今のところは、いくつかの未解決の問題を回避する必要があります。

生成されたページの内容は、現在のところ、生成時に定義した実際のモデル属性を
使用していません。すぐに使用するようになりますが、それまでの間は生成された
ページを修正していきましょう。

### 質問ページ {#question-pages}

[//]:

  "`generate` が実際のモデル属性を使用する場合は、以下のセクションを削除しましょう"

`app/pages/questions/index.tsx` にジャンプしてください。 `QuestionsList`
コンポーネントが生成されていることに注意してください。

```tsx
// app/pages/questions/index.tsx

export const QuestionsList = () => {
  const router = useRouter()
  const page = Number(router.query.page) || 0
  const [{questions, hasMore}] = usePaginatedQuery(getQuestions, {
    orderBy: {id: "asc"},
    skip: ITEMS_PER_PAGE * page,
    take: ITEMS_PER_PAGE,
  })

  const goToPreviousPage = () => router.push({query: {page: page - 1}})
  const goToNextPage = () => router.push({query: {page: page + 1}})

  return (
    <div>
      <ul>
        {questions.map((question) => (
          <li key={question.id}>
            <Link href={`/questions/${question.id}`}>
              <a>{question.name}</a>
            </Link>
          </li>
        ))}
      </ul>

      <button disabled={page === 0} onClick={goToPreviousPage}>
        Previous
      </button>
      <button disabled={!hasMore} onClick={goToNextPage}>
        Next
      </button>
    </div>
  )
}
```

しかし、これではうまくいきません。上で作成した `Question` モデルには
`name` フィールドがないことを覚えておいてください。これを修正するには
、`question.name` を `question.text` に置き換えます。

```diff
// app/pages/questions/index.tsx

export const QuestionsList = () => {
  const router = useRouter()
  const page = Number(router.query.page) || 0
  const [{questions, hasMore}] = usePaginatedQuery(getQuestions, {
    orderBy: {id: "asc"},
    skip: ITEMS_PER_PAGE * page,
    take: ITEMS_PER_PAGE,
  })

  const goToPreviousPage = () => router.push({query: {page: page - 1}})
  const goToNextPage = () => router.push({query: {page: page + 1}})

  return (
    <div>
      <ul>
        {questions.map((question) => (
          <li key={question.id}>
            <Link href={`/questions/${question.id}`}>
-              <a>{question.name}</a>
+              <a>{question.text}</a>
            </Link>
          </li>
        ))}
      </ul>

      <button disabled={page === 0} onClick={goToPreviousPage}>
        Previous
      </button>
      <button disabled={!hasMore} onClick={goToNextPage}>
        Next
      </button>
    </div>
  )
}
```

次に、同様の修正を `app/questions/components/QuestionForm.tsx` にも適用し
てみましょう。フォームの投稿において、`LabeledTextField` の `name` を
`"text"` に置き換えます。

```diff
export function QuestionForm<S extends z.ZodType<any, any>>(
  props: FormProps<S>,
) {
  return (
    <Form<S> {...props}>
-     <LabeledTextField name="name" label="Name" placeholder="Name" />
+     <LabeledTextField name="text" label="Text" placeholder="Text" />
    </Form>
  )
}
```

### `deleteQuestion` Mutation {#delete-question-mutation}

[//]: # "Prismaがカスケード削除をサポートしたら、このセクションを削除する"

Prisma はまだ "カスケード削除 "をサポートしていません。このチュートリアル
では、`Question` を削除する際に `Choice` データを削除しないことを意味しま
す。これを手動で行うためには、生成された `deleteQuestion` Mutation を一時
的に拡張する必要があります。テキストエディタで
`app/questions/mutations/deleteQuestion.ts` を開き、関数本体の先頭に以下を
追加してください。

```ts
await db.choice.deleteMany({where: {questionId: id}})
```

最終的には次のようになるはずです。

```diff
// app/questions/mutations/deleteQuestion.ts

export default resolver.pipe(
  resolver.zod(DeleteQuestion),
  resolver.authorize(),
  async ({id}) => {
+   await db.choice.deleteMany({where: {questionId: id}})
    const question = await db.question.deleteMany({where: {id}})

    return question
  },
)
```

この Mutation は、問題自体を削除する前に、問題に関連する選択肢を削除するよ
うになりました。

## 質問フォームに選択肢を追加する {#adding-choices-to-the-question-form}

お疲れ様です！ 次は質問フォームに選択肢を追加します。エディタで
`app/questions/components/QuestionForm.tsx` を開いてください。

さらに、3 つの選択肢 `<LabeledTextField>` コンポーネントを追加します。

```diff
export function QuestionForm<S extends z.ZodType<any, any>>(
  props: FormProps<S>,
) {
  return (
    <Form<S> {...props}>
      <LabeledTextField name="text" label="Text" placeholder="Text" />
+     <LabeledTextField name="choices.0.text" label="Choice 1" />
+     <LabeledTextField name="choices.1.text" label="Choice 2" />
+     <LabeledTextField name="choices.2.text" label="Choice 3" />
    </Form>
  )
}
```

次に `app/pages/questions/new.tsx` を開き、`initialValues` を以下のように
設定します。

```diff
      <QuestionForm
        submitText="Create Question"
-       // initialValues={{ }}
+       initialValues={{choices: []}}
        onSubmit={async (values) => {
          try {
            const question = await createQuestionMutation(values)
            router.push(`/questions/${question.id}`)
          } catch (error) {
            console.error(error)
            return {
              [FORM_ERROR]: error.toString(),
            }
          }
        }}
      />
```

それから `app/questions/mutations/createQuestion.ts` を開き、Mutation で選
択肢のデータが受け入れられるように zod スキーマを更新します。また、選択肢
が作成されるように `db.question.create` コールを更新する必要があります。

```diff
// app/questions/mutations/createQuestion.ts

const CreateQuestion = z
  .object({
    text: z.string(),
+   choices: z.array(z.object({text: z.string()})),
  })
  .nonstrict()

export default resolver.pipe(
  resolver.zod(CreateQuestion),
  resolver.authorize(),
  async (input) => {
-   const question = await db.question.create({data: input})
+   const question = await db.question.create({
+     data: {
+       ...input,
+       choices: {create: input.choices},
+     },
+   })

    return question
  },
)
```

#### お試し

これで、`localhost:3000/questions/new` を開いて、新しい問題と選択肢を作成
することができます。

## リストの選択 {#listing-choices}

休憩です。ブラウザの `localhost:3000/questions` に戻って、作成したすべての
問題を見てください。ここに質問の選択肢をリストアップしてみましょう。 まず
、質問のクエリをカスタマイズする必要があります。Prisma では、ネストされた
関係をクエリしたいことをクライアントに手動で知らせる必要があります。あなた
の `getQuestion.ts` と `getQuestions.ts` ファイルを以下のように変更してく
ださい。

```diff
// app/questions/queries/getQuestion.ts

const GetQuestion = z.object({
  // This accepts type of undefined, but is required at runtime
  id: z.number().optional().refine(Boolean, "Required"),
})

export default resolver.pipe(
```

```tsx
export const Question = () => {
  const router = useRouter()
  const questionId = useParam("questionId", "number")
  const [deleteQuestionMutation] = useMutation(deleteQuestion)
  const [question, {refetch}] = useQuery(getQuestion, {id: questionId})

      await updateChoiceMutation({id})

    }
  }

    <>
      <Head>
        <title>Question {question.id}</title>
      </Head>

      <div>
        <h1>{question.text}</h1>

          {question.choices.map((choice) => (
            <li key={choice.id}>
              {choice.text} - {choice.votes} votes
              <button onClick={() => handleVote(choice.id)}>Vote</button>

          <a>Edit</a>
        </Link>
```

## 最後に、既存の質問の編集の選択肢を許可しよう {#edit-choices-for-question}

既存の問題の **Edit** ボタンをクリックすると、問題を作成するのと同じフォー
ムを使用していることがわかります。これですでに完了です。あとは Mutation を
更新するだけです。

`app/questions/mutations/updateQuestion.ts` を開き、以下の変更を行ってくだ
さい。

```diff
// app/questions/mutations/updateQuestion.ts
import {resolver} from "blitz"
import db from "db"
import * as z from "zod"

const UpdateQuestion = z
  .object({
    id: z.number(),
    text: z.string(),
+   choices: z.array(
+     z.object({id: z.number().optional(), text: z.string()}).nonstrict(),
+   ),
  })
  .nonstrict()

export default resolver.pipe(
  resolver.zod(UpdateQuestion),
  resolver.authorize(),
  async ({id, ...data}) => {
-   const question = await db.question.update({where: {id}, data})
+   const question = await db.question.update({
+     where: {id},
+     data: {
+       ...data,
+       choices: {
+         upsert: data.choices.map((choice) => ({
+           // Appears to be a prisma bug,
+           // because `|| 0` shouldn't be needed
+           where: {id: choice.id || 0},
+           create: {text: choice.text},
+           update: {text: choice.text},
+         })),
+       },
+     },
+   })

    return question
  },
)
```

[`upsert`](https://www.prisma.io/docs/reference/api-reference/prisma-client-reference#upsert)
は、「この項目が存在すればそれを更新する、存在しなければ作成する」という意
味です。質問を作成する際にユーザーが 3 つの選択肢を追加する必要がないので
、このケースには最適です。後でユーザーが質問を編集して別の選択肢を追加した
場合、ここに作成されます。
